## ┬¦0 ŌĆö Correction note (9 May 2026)

The original draft of this inventory made a systematic error in evaluating kupua status for ES-only filter params.

**The error:** For params like `takenSince`, `modifiedSince`, `uploadedBy`, `archived`, `countAll`, and the date-field selector, the evaluating agent asked "does kupua call the media-api with this param?" ŌĆö but kupua is **direct-ES**, not API-first. The correct question is "does kupua's `es-adapter.ts` / `searchParamsSchema` / `DateFilter.tsx` handle this?"

**Corrections made:**
- `takenSince`/`takenUntil`: Don't-have ŌåÆ **Have-full** (in schema, adapter, DateFilter UI)
- `modifiedSince`/`modifiedUntil`: Don't-have ŌåÆ **Have-full** (in schema, adapter, DateFilter UI)
- `uploadedBy`: Don't-have ŌåÆ **Have-partial** (schema + adapter + CQL `uploader:` alias; no "My uploads" checkbox UI)
- `archived`: Don't-have ŌåÆ **Have-partial** (CQL `archived:true/false` via EDITS_FIELDS; no URL param)
- Date field selector: Have-partial ŌåÆ **Have-full** (DateFilter.tsx implements all three modes)
- `countAll`: Don't-have ŌåÆ **Have-full** (internal to search store; deliberately not a URL param)
- `labelSearch` endpoint: Clarified ŌĆö this is the **autocomplete API** for label typeahead suggestions, NOT label search itself. `label:foo` CQL search works in kupua via the CQL translator and was never missing. The autocomplete endpoint (calling media-api `label-search` rel) is genuinely absent.

**Second correction (9 May 2026):** `has:crops` works in kupua ŌĆö and so does every other URL-param filter in the Both section. The same evaluation error recurred: the subagent evaluated against "does kupua call media-api with this?" rather than "is it in the DAL/schema/store?" All four Both-section filter rows corrected:
- `hasCrops`: Don't-have ŌåÆ **Have-full** (`searchParamsSchema`, `es-adapter.ts:234-236`, `search-store.ts`; CQL `has:crops` also works via `FIELD_ALIASES["crops"]`)
- `hasRightsAcquired`: Don't-have ŌåÆ **Have-full** (`searchParamsSchema`, `es-adapter.ts:241-245`)
- `persisted`: Don't-have ŌåÆ **Have-full** (`searchParamsSchema`, `es-adapter.ts:261-265`)
- `syndicationStatus` was already Have-partial in inventory A (enrichment overlay); the URL param is also in the schema and adapter ŌĆö consistent.
- Typeahead fields: Don't-have ŌåÆ **Have-full**. Kupua has a complete, purpose-built typeahead system (`typeahead-fields.ts`, `lazy-typeahead.ts`, `CqlSearchInput.tsx`) covering the same field set as kahuna and improving on several points (ES agg resolvers instead of media-api calls; fixes kahuna's eager-evaluation bug; adds resolvers for `label`/`photoshoot`/`keyword`/`croppedBy` that kahuna lacks). Minor residual gap: no value resolver for `by` (photographer list); `usages@platform` missing BBC "Added to Photo Sales" option.
